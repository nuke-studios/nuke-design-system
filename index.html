The Holy Grail: Universal .style-1/2/3 Pattern

Same classes, different meanings per element

This is what makes Nuke special - no naming fatigue!

button.style-1
Solid button
button.style-2
Outlined button
button.style-3
Ghost button
input.style-1
Border input
input.style-2
Filled input
input.style-3
Underline input
checkbox.style-1
Square checkbox
checkbox.style-2
Rounded checkbox
checkbox.style-3
Circle checkbox
table.style-1
Striped table
table.style-2
Bordered table
table.style-3
Minimal table

Why this is brilliant:

  • No mental overhead - just numbers
  • Consistent naming across all elements
  • Easy to remember and teach
  • No BEM, no utility classes, no complexity
  • Native HTML elements work without classes
  • Add .style-1/2/3 for variants

Architecture: Core + Theme Separation

Clean separation of logic and design

Customize the theme without touching core files

Core (Logic)

Pure CSS that uses variables but never defines them:

  • core/reset.css
  • core/animations.css
  • core/typography.css
  • core/ui/button.css
  • core/ui/input.css
  • ...all UI elements

â†’ Installed in node_modules
â†’ Never modified by users
â†’ Can be safely updated

Theme (Variables)

All design tokens and customization:

  • theme/colors.css
  • theme/spacing.css
  • theme/typography.css
  • theme/button.css
  • theme/input.css
  • ...all element variables

â†’ Copied to project root
â†’ Fully customizable
â†’ Never overwritten on updates

Your project setup (style.css):

/* Import theme FIRST (variables) */
@import 'theme/all.css';

/* Then import core (logic) */
@import 'core/all.css';

/* Your custom styles */
body {
  font-family: your-font;
}
